# SYSTEM SPECIFICATION: SMART MEAL PLANNER & GROCERY APP
## Version 2.2 — Updated Master Prompt

---

## 1. PROJECT OVERVIEW

A personal web application for meal planning, diet tracking, and smart grocery list generation — designed for **seamless use on both smartphone and desktop**.

The app runs on a **weekly cycle**: planning happens on **Friday**, shopping on **Saturday morning**. The meal plan covers **Saturday to Friday**.

### Tech Stack (responsive, device-agnostic)
- **Backend:** FastAPI (Python) — lightweight, fast, REST API
- **Frontend:** React (Vite) with Tailwind CSS — fully responsive PWA, optimized for both mobile and desktop
- **Database:** SQLite via SQLAlchemy — local, file-based, auto-initialized
- **Nutritional data:** Open Food Facts API or USDA FoodData Central — auto-filled on ingredient entry
- **Deployment:** Runs locally or on a VPS/Raspberry Pi; accessible via browser on any device

> The app must function smoothly as a **Progressive Web App (PWA)** so it can be added to the home screen on mobile and used as a full-featured planning tool on desktop.

## 2. CORE FEATURES & ROADMAP

### PHASE 1: Core CRUD & Weekly Planner *(Foundation)*

**Recipe Database**
- Add recipes with: title, instructions, source (URL or note), number of portions, meat dish (boolean), prep time (minutes), season tags
- Ingredients per recipe: name, quantity, unit, category
- Nutritional values **auto-filled** via food database API (calories, carbs, protein, fat per 100g → calculated per portion)
- Import via **JSON paste** (generated by Claude from URL or screenshot)

**Portion Management**
- Every recipe stores a base portion count (e.g., "for 4 persons")
- When adding to meal plan, user selects **how many portions** they need
- All ingredient quantities and nutritional values scale automatically

**Weekly Planner Matrix**
- Grid: Saturday → Friday, 6 meal slots per day:
  1. Breakfast
  2. Snack 1
  3. Lunch
  4. Snack 2
  5. Dinner
  6. Evening Snack
- Each slot: assign a recipe (with portion count) or a simple manual item
- Daily tracker: check off each slot as "Eaten ✓"

**Full Edit Flexibility on the Planned Menu**
- Any planned slot can be **edited at any time** after the initial planning — not just checked off
- Actions per slot:
  - **Replace**: swap the current recipe for a different one
  - **Clear**: remove a recipe from a slot without replacing it
  - **Adjust portions**: change the number of portions planned
  - **Swap slots**: exchange two slots across days or within the same day (e.g., swap Friday lunch with Saturday lunch) — via drag-and-drop on desktop, or a dedicated "swap" action on mobile
- When a slot is edited or swapped, the grocery list and nutritional dashboard **update automatically**

**Smart Grocery List**
- Auto-aggregates all ingredients from the weekly plan
- Intelligently **merges duplicate ingredients** across recipes (e.g., 2× onion from different recipes → combined into one line)
- Manually add non-recipe items (household products, etc.)
- Automatically **sorted by supermarket aisle category** (consistent order = efficient shopping route)
- Items can be checked off while shopping

### PHASE 2: Diet Parameters & Constraints

**User Profile / Daily & Weekly Targets**
- Max. daily calories (kcal) — default 1900 kcal, freely adjustable at any time (e.g. when diet evolves)
- Target macro split: % Carbs / % Protein / % Fats
- Max. number of meat dishes per week
- All targets are stored per profile and can be updated independently without affecting historical data

**Live Planning Dashboard (while composing the week)**
- Visual progress bars per day (planned calories vs. max) and per week (meat count)
- Real-time warnings when constraints are exceeded while planning
- Overview of nutritional balance across the planned week

**Daily Nutritional Tracker (based on confirmed eaten meals)**
- Values only update when a meal slot is marked as **"Eaten ✓"** — planned but not yet eaten meals are shown separately as "upcoming"
- **Macro pie chart**: a circular/donut chart showing the proportion of Carbohydrates / Proteins / Fats consumed so far that day, updating with each confirmed meal
- **Calorie gauge**: a graphical indicator (e.g. a filled arc or radial progress bar) showing:
  - Calories consumed so far (confirmed eaten)
  - Calories remaining until the daily maximum
  - Clear visual distinction between "safe zone", "approaching limit", and "exceeded"
- Both visuals are prominently displayed on the **Today view** so they are the first thing seen when opening the app
- A daily summary is also accessible in the **Week view** per day column

---

### PHASE 3: Food Waste Reduction & Leftover Optimizer

**Partial Ingredient Tracking**
- When a recipe uses only part of an ingredient (e.g., ½ cauliflower), the app registers the remainder as a "leftover in stock"
- Leftovers are tracked with an estimated expiry window

**Smart Suggestions**
- Filter: *"Show recipes that use my current leftovers"*
- When suggesting meals, the app prioritizes recipes that use up leftover ingredients
- This helps compose a balanced, waste-free menu without extra shopping

---

### PHASE 4: Seasonal Awareness

**Seasonal Ingredient Tags**
- Each ingredient/recipe can be tagged with one or more seasons: Spring / Summer / Autumn / Winter
- When browsing or receiving suggestions, seasonal recipes for the current period are highlighted
- Optional filter: *"Show only seasonal recipes"*
- Helps make healthier, cheaper, and more sustainable choices

## 3. AI-ASSISTED SUGGESTIONS

When the user has little inspiration or wants help composing a balanced week:

- **"Suggest my week"** button: the app proposes a full weekly menu based on:
  - Active diet constraints (calories, meat limit)
  - Current season
  - Available leftovers
  - Recipes not used recently (variety)
- Individual slot suggestions: *"What can I eat for dinner on Tuesday?"*
- Suggestions respect partial ingredient stock to minimize waste

## 4. EXTERNAL INPUT WORKFLOW (Links & Screenshots)

Parsing URLs and OCR on screenshots is **not handled natively** by the app — this keeps the codebase lightweight.

**Workflow:**
1. User pastes a recipe URL or screenshot into a **separate Claude chat**
2. Claude converts it into structured JSON (see format below)
3. User copies the JSON and pastes it into the app's **"Import Recipe (JSON)"** field
4. App parses and saves it to the database, auto-fetching nutritional values

**Standard JSON Recipe Format:**
```json
{
  "title": "Bloemkoolsoep",
  "source": "https://...",
  "portions": 4,
  "prep_time_minutes": 30,
  "meat_dish": false,
  "seasons": ["autumn", "winter"],
  "instructions": "...",
  "ingredients": [
    { "name": "bloemkool", "quantity": 0.5, "unit": "stuk", "category": "Groenten" },
    { "name": "room", "quantity": 200, "unit": "ml", "category": "Zuivel" }
  ]
}
```

## 5. DATABASE STRUCTURE (SQLite / SQLAlchemy)

| Table | Key Fields |
|---|---|
| `recipes` | id, title, source, instructions, portions, prep_time_min, meat_dish (bool), seasons (JSON array) |
| `ingredients` | id, recipe_id, name, quantity, unit, category_id |
| `nutritional_values` | id, ingredient_id, kcal_per_100g, carbs_g, protein_g, fat_g (auto-filled via API) |
| `categories` | id, name (e.g. Groenten, Zuivel, Vlees, Huishouden), sort_order |
| `meal_plan` | id, date, slot (1–6), recipe_id, portions_planned, eaten (bool) |
| `leftovers` | id, ingredient_name, quantity, unit, estimated_expiry_date |
| `manual_groceries` | id, week_start_date, name, category_id, checked (bool) |
| `user_targets` | id, max_kcal_per_day (default 1900), max_meat_per_week, target_carbs_pct, target_protein_pct, target_fats_pct, valid_from_date |

## 6. UX & RESPONSIVE DESIGN REQUIREMENTS

**General**
- **PWA**: installable on Android/iOS home screen, works offline for viewing current week plan
- **Fully responsive**: layouts adapt fluidly between mobile (≤768px), tablet, and desktop (≥1024px)
- **Today view**: opens by default — shows today's 6 slots at a glance with check-off and edit capability
- **Grocery list**: grouped by category with clear dividers; checkboxes easy to tap or click
- **Fast recipe search**: search + filter by season, prep time, meat/no meat

**Mobile (smartphone)**
- Bottom navigation bar: max. 4–5 tabs (Today / Week / Recipes / Groceries / Profile)
- Large tap targets for all interactive elements (min. 44×44px)
- Slot editing via contextual action sheet (tap slot → options: Replace / Clear / Swap / Adjust portions)
- Swap slots via a two-step "Swap" action: select source slot → select target slot

**Desktop**
- Sidebar navigation instead of bottom bar
- Full weekly grid visible at once (7 columns × 6 rows) for planning overview
- Drag-and-drop to swap or move meal slots between days/positions
- Inline editing by clicking a slot directly
- Wider recipe import and nutritional dashboard panels
